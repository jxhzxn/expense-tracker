What Was Built

🏠 Home Screen

Shows your total spent and a large "Add Expense" button. Acts as the app's dashboard.

💰 Add Amount

Step 1 of 2. A focused numeric input where you type the amount you spent.

🏷️ Select Category

Step 2 of 2. Tap a category from a grid — the expense saves immediately and you're returned home.

📋 Expense History

A sortable list of all your saved expenses. Supports 4 sort modes. Shows an empty state when no expenses exist.

Navigation Architecture
NavigationContainer — in App.tsx
└─ RootNavigator (Bottom Tabs)
├─ Tab: Home → AddExpenseStack (Native Stack)
├─ HomeScreen
├─ AddAmountScreen
└─ AddCategoryScreen receives amount as route param
└─ Tab: History
└─ HistoryScreen

The add-expense flow lives inside the Home tab so the bottom tab bar disappears during the 2-step entry — preventing the user from accidentally tabbing away mid-flow.

State Management

Context  ExpenseContext.tsx

Global state is managed with React Context + useReducer. No external library needed. The reducer handles two actions:

  • LOAD_EXPENSES — called on app startup to hydrate state from AsyncStorage
  • ADD_EXPENSE — adds a new expense to the top of the list

Two useEffect hooks handle persistence automatically — one loads data on mount, the other saves whenever the expenses array changes. No manual save calls needed anywhere in the app.

Future actions (e.g. DELETE_EXPENSE, EDIT_EXPENSE) can be added to the reducer without restructuring anything.

Storage  storage.ts

Two plain async functions that read/write a JSON array to AsyncStorage under the key @expense_tracker_expenses. Pure functions — easy to test and replace with an API later.

Sort state lives in HistoryScreen local state (not in global context) — it's a view preference, not shared data. The sorted list is computed with useMemo so it only re-calculates when expenses or the sort option change.

Tech Stack & Libraries
expo ~54

App runtime and dev tooling. Lets you run on iOS/Android without Xcode/Android Studio via Expo Go.

react-native 0.81.5

Core framework. All UI is built with native components (View, Text, FlatList, etc.).

@react-navigation/native

Core navigation library. Provides NavigationContainer and navigation hooks.

@react-navigation/bottom-tabs

Renders the Home / History tab bar at the bottom of the screen.

@react-navigation/native-stack

Manages the Home → AddAmount → AddCategory navigation stack.

@react-native-async-storage

Local key-value storage. Persists expenses between app restarts — no backend needed.

react-native-uuid

Generates unique IDs (UUID v4) for each expense without needing a server.

typescript 5.9

Full type safety across all files. TypeScript check passes with zero errors.

How to Run
1

Load Node.js (nvm)

Node.js is installed via nvm. Load it in your terminal before running any commands.

export NVM_DIR="$HOME/.nvm" && \. "$NVM_DIR/nvm.sh"
2

Start Expo dev server

cd "/Users/jahzan/Documents/Projects/Expense Tracker"
npx expo start
3a

Run on your iPhone (easiest)

Install Expo Go from the App Store, then scan the QR code shown in the terminal.

3b

Run on iOS Simulator

Requires the full Xcode app (free on Mac App Store, ~15 GB). After installing Xcode, press i in the terminal while Expo is running.
